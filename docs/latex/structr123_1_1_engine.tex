\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename C\+B\+R\+NG$>$\newline
struct r123\+::\+Engine$<$ C\+B\+R\+N\+G $>$}

If G satisfies the requirements of a C\+B\+R\+NG, and has a ctr\+\_\+type whose value\+\_\+type is an unsigned integral type, then Engine$<$\+G$>$ satisfies the requirements of a C++0x \char`\"{}\+Uniform Random Number Engine\char`\"{} and can be used in any context where such an object is expected.

Note that wrapping a counter based R\+NG with a traditional A\+PI in this way obscures much of the power of counter based P\+R\+N\+Gs. Nevertheless, it may be of value in applications that are already coded to work with the C++0x random number engines.

The \mbox{\hyperlink{classr123_1_1_micro_u_r_n_g}{Micro\+U\+R\+NG}} template in \mbox{\hyperlink{_micro_u_r_n_g_8hpp_source}{Micro\+U\+R\+N\+G.\+hpp}} provides the more limited functionality of a C++0x \char`\"{}\+Uniform
\+Random Number Generator\char`\"{}, but leaves the application in control of counters and keys and hence may be preferable to the \mbox{\hyperlink{structr123_1_1_engine}{Engine}} template. For example, a \mbox{\hyperlink{classr123_1_1_micro_u_r_n_g}{Micro\+U\+R\+NG}} allows one to use C++0x \char`\"{}\+Random Number
\+Distributions\char`\"{} without giving up control over the counters and keys.
